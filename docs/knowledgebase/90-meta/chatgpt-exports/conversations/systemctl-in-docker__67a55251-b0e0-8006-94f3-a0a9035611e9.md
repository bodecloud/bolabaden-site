# Systemctl in Docker

Source: [ChatGPT conversation](https://chatgpt.com/c/67a55251-b0e0-8006-94f3-a0a9035611e9)

## User

in Dockerfile i'm using a ubuntu image and trying to install and use systemctl. How to fix this error:

--> RUN systemctl daemon-reload
System has not been booted with systemd as init system (PID 1). Can't operate.
Failed to connect to bus: Host is down

I've already sudo apt install systemd

Be concise and accurate

NOTE: Literally the only thing i can modify is Dockerfile. I can't modify the command that runs Dockerfile.
Show more

## Assistant

Your error occurs because Ubuntu’s official Docker image uses init as PID 1 instead of systemd. Docker containers don’t boot like a full OS, so systemctl won’t work by default.

To fix this within the Dockerfile, you need to:

Use systemd as the entrypoint.

Enable systemd within the container.

Solution: Modify your Dockerfile to:
dockerfile
FROM ubuntu:latest  
RUN apt update && apt install -y systemd  
CMD ["/bin/bash", "-c", "exec /lib/systemd/systemd"]  

This ensures the container starts with systemd, allowing systemctl commands to work.

## User

don't have /bin/bash. Your solution also cannot use CMD, that's reserved for something else.

## Assistant

Since you can't modify the command that runs the Dockerfile and don't have /bin/bash, you'll need a workaround to simulate systemctl functionality. Try replacing systemctl with dbus-run-session:

Solution:

Modify your Dockerfile to install dbus and use dbus-run-session for systemctl commands:

dockerfile
FROM ubuntu:latest  
RUN apt update && apt install -y systemd dbus  
RUN dbus-run-session -- systemctl daemon-reload

This will allow systemctl commands to work without needing systemd as PID 1.

## User

whats the --

## Assistant

The -- in dbus-run-session -- systemctl daemon-reload separates dbus-run-session options from the command being executed. In this case, dbus-run-session doesn't require additional options, so -- just ensures that systemctl daemon-reload is interpreted correctly as the command to run within the session.
