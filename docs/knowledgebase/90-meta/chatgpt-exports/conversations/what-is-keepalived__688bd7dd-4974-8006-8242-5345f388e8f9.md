# What is Keepalived

Source: [ChatGPT conversation](https://chatgpt.com/c/688bd7dd-4974-8006-8242-5345f388e8f9)

## User

what is keepalived

## Assistant

Keepalived is a routing software written in C that provides high availability (HA) and load balancing primarily for Linux systems. It's most commonly used with Linux Virtual Server (LVS) to create redundant and fault-tolerant network services. Here's a breakdown of what it does and why it's useful:

🔧 Main Features

High Availability (HA):

Keepalived uses VRRP (Virtual Router Redundancy Protocol) to assign a virtual IP address (VIP) to a group of servers.

If the primary node (master) fails, a backup node takes over the VIP, ensuring minimal service disruption.

Load Balancing:

Can work in conjunction with IPVS (IP Virtual Server) to distribute traffic among backend servers.

Health Checking:

Monitors the health of services (like HTTP, MySQL, etc.) using built-in or custom health checks.

If a service fails on a node, it can be taken out of the load-balancing pool or failover can be triggered.

🖥️ Common Use Cases

Failover between two NGINX or HAProxy servers (e.g., active-passive setups).

Load balancing TCP/UDP traffic across a cluster.

Providing VIPs for Kubernetes ingress controllers or databases.

🧱 How It Works (Simplified)

Nodes in a cluster run Keepalived.

One is elected MASTER, others are BACKUP.

All share a virtual IP (VIP).

When the master fails, a backup takes over the VIP using VRRP.

You're running a web service behind NGINX on two servers:

Use Keepalived to create a VIP (e.g., 192.168.1.100)

Clients connect to this VIP.

If NGINX on Server A fails, Server B automatically takes over the VIP — seamless failover.
