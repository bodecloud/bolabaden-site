# K3s Factory Reset

Source: [ChatGPT conversation](https://chatgpt.com/c/6838b246-8248-8006-b162-6ac68b0ae890)

## User

write a kubectl command that'll restore factory defaults of my k3s

## Assistant

There is no single kubectl command that resets a K3s (Lightweight Kubernetes) cluster to factory defaults, because kubectl interacts with the Kubernetes API, not the K3s service or its files directly.

To restore factory defaults of a K3s cluster, you need to reset the cluster by stopping K3s and deleting its data.

⚠️ This is destructive and will erase everything (cluster state, workloads, etc).
🧨 Manual Reset Steps (for K3s installed via script):

Run this on your K3s server node(s):

Bash
sudo systemctl stop k3s
sudo rm -rf /etc/rancher/k3s /var/lib/rancher/k3s /var/lib/kubelet /etc/cni /opt/cni /var/lib/cni
sudo ip link delete cni0
sudo ip link delete flannel.1
sudo systemctl start k3s

If you’re running k3s agent on worker nodes, use k3s-agent instead of k3s in the commands.

Optional: If you're using kubectl, you may want to also remove the kubeconfig file:
Bash
rm ~/.kube/config
Reinstall K3s (optional):

If you want a totally fresh install:

Bash
curl -sfL https://get.k3s.io | sh -
