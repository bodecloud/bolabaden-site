1. How to Add Automatic Subdomains for Containers

To automatically resolve container names to their Kubernetes Service IPs, modify your Corefile like this:

Updated Corefile for CoreDNS
YAML
apiVersion: v1
data:
  Corefile: |
    .:53 {
        errors
        health {
           lameduck 5s
        }
        ready
        kubernetes cluster.local in-addr.arpa ip6.arpa {
           pods insecure
           fallthrough in-addr.arpa ip6.arpa
           ttl 30
        }
        prometheus :9153
        forward . /etc/resolv.conf {
           max_concurrent 1000
        }
        cache 30 {
           disable success cluster.local
           disable denial cluster.local
        }
        loop
        reload
        loadbalance
    }

    # Automatically create subdomains for all services
    myprivatedomain.local {
        errors
        log
        hosts {
            10.43.0.10 app1.myprivatedomain.local
            10.43.0.11 app2.myprivatedomain.local
            fallthrough
        }
    }
Explanation

✅ myprivatedomain.local → Your custom private domain.
✅ The hosts plugin → Manually maps Kubernetes service IPs to subdomains.
✅ The fallthrough directive → Ensures it doesn't break other DNS queries.

2. How to Automatically Map All Containers

Alternative: Use Kubernetes-Based Resolution

http://myapp.default.svc.cluster.local
