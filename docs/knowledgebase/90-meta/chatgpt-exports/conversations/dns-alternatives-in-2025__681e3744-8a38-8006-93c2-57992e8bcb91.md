Here's how it looks in IPv4:
[ IP HEADER ]
 ├─ Version
 ├─ Source IP
 ├─ Destination IP

 └─ (more...)

[ PAYLOAD ]
 └─ Whatever header your protocol type expects (e.g. TCP header)

So when you send a packet, you're literally:

Assigning a number that says “I’m a TCP packet” or “I’m a UDP packet.”

Filling out a header format that matches.

🧠 That’s it. That’s the difference.

💥 Can I just send my own arbitrary packet?
✅ Yes — with raw sockets (on Linux, BSD, etc.)

You can bypass the kernel’s high-level TCP/UDP stack and manually craft a packet:

Build your own IP headers.

Set the protocol number to something custom, like 42.

Fill in whatever fake header or garbage data you want.

These are called raw packets.
