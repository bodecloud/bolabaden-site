## Assistant

Ah! This error is actually coming from Traefik v3’s Docker provider:

error="field not found, node: failover" ...

In Traefik v3, there is no failover field for routers or services. The syntax you are trying (traefik.http.routers.whoami.failover.service) is from Traefik v2.x experimental features or old configurations. Traefik v3 removed the failover option entirely. That’s why both router.failover and service.failover produce the same error.

How to fix

You need to remove any failover labels entirely. Instead, you should:

Define multiple servers under a file provider service if you want load balancing or failover.

Point the router directly to that service.

So in your Docker labels:

YAML
labels:
  traefik.enable: "true"

  # Router for normal access
  traefik.http.routers.whoami.rule: 'Host(`whoami.${DOMAIN}`)'
  traefik.http.routers.whoami.service: whoami-servers@file  # point directly to file-defined service

  # Direct access router
  traefik.http.routers.whoami-direct.rule: 'Host(`whoami.${TS_HOSTNAME}.${DOMAIN}`)'
  traefik.http.routers.whoami-direct.service: whoami@docker
  traefik.http.routers.whoami-direct.middlewares: nginx-auth@docker

  # Service definition for Docker container
  traefik.http.services.whoami.loadbalancer.server.port: 80

Notice that failover is completely gone. Instead, the whoami-servers@file service (from your file provider) handles multiple backends.
