- **Transformer-Based Embeddings Latency**: In contrast, transformer-based embeddings paired with a vector database like ChromaDB can retrieve relevant information in **milliseconds**. For example:
  - On local hardware (even without GPUs), transformer-based embeddings can achieve retrieval times of around **10-50 milliseconds** per query for datasets with millions of embeddings, thanks to ANN search algorithms.

### **Can SDM Run on Local Hardware?**
Yes, SDM can run on local hardware, but it will be much slower than transformer-based methods:
- On a modern CPU, SDM can handle small datasets but will experience significant slowdowns as the number of stored patterns increases.
- For practical purposes (e.g., real-time chatbot interactions), SDM would likely be too slow unless heavily optimized or used with specialized hardware.

### **Conclusion**
In terms of raw speed:
- **SDM is significantly slower**, often taking seconds or minutes per query as it scales poorly with data size.
- **Transformer-based embeddings are much faster**, typically taking milliseconds per query even for large datasets.

For real-time applications like chatbots, transformer-based embeddings are far more efficient and scalable than SDM.
