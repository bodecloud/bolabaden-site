Regarding PySide, here are some key points from the search results:

1. PySide was initially developed by Nokia in 2009 when they owned Qt, as an alternative to PyQt with a more permissive license.[3]

2. PySide is now developed by The Qt Company and is considered the official set of Python bindings for Qt.[4]

3. PySide is distributed under the LGPL license, which makes it easier to incorporate into commercial projects compared to PyQt's GPL license.[4]

4. The current stable version of PySide6 is 6.1.2 as of the information in the search results.[6]

5. PySide allows the use of QtQuick or QML to establish the user interface and supports primary widget-based user interface resources.[4]

6. One disadvantage of PySide mentioned is the lack of proper documentation, which can make it difficult to learn all its features.[4]

7. The Qt for Python project (which includes PySide) aims to provide a complete port of the PySide module to Qt. Development started on GitHub in May 2015.[5]

8. PySide6 was released for Qt6 in December 2020, dropping support for Python 2.7 and older versions of Python 3.[5]

While PySide and PyQt are very similar in terms of API, the main difference lies in their licensing models, with PySide being more permissive for commercial use without requiring the purchase of a license.
