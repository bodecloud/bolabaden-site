## Integrating LilyPond with Python for Music Applications

Research suggests that LilyPond can be effectively integrated into Python projects, such as an audio editor or mixer app, by using libraries to generate or manipulate LilyPond code and then compiling it to produce sheet music or MIDI files. The evidence leans toward using PyPI packages for seamless installation, avoiding manual downloads for end users, though some setups may require additional tools for audio rendering.

- **Key Integration Options**: Libraries like python-ly and Abjad allow generating or editing LilyPond (.ly) files in Python code. For compilation without separate installation, the 'lilypond' PyPI package bundles the executable.
- **Availability on PyPI**: Yes, packages such as 'lilypond', 'python-ly', and 'abjad' are available, enabling pip installation for end users.
- **Audio Workflow**: LilyPond generates MIDI, which can be converted to audio formats for mixing; this may involve additional libraries or tools.

### Recommended Approach
For an audio editor/mixer app, start by using a Python library to create or modify LilyPond code programmatically. Compile the .ly file to MIDI using the bundled LilyPond executable from the 'lilypond' package. Then, render the MIDI to audio (e.g., WAV) using a synthesizer like FluidSynth (via subprocess) or Python libraries if available. Mix audio tracks with libraries like pydub. This setup keeps everything Python-based and installable via pip.

Install via pip:
- `pip install lilypond python-ly` (or abjad for more advanced composition).

Example basic usage:
```python
import subprocess
import lilypond  # Provides the executable path

# Generate a simple .ly file
ly_code = r"""
\version "2.24.0"
\score {
  \new Staff { c'4 d' e' f' }
  \midi { }
}
"""
with open("example.ly", "w") as f:
    f.write(ly_code)

# Compile to MIDI
subprocess.run([lilypond.executable(), "--output=example", "example.ly"])
```
This produces `example.midi`, which you can then process in your app.

- **Dependencies**: Ensure Python 3.10+ for compatibility with LilyPond requirements.

### Building an Audio Editor/Mixer App
In an app context:
1. **Generate LilyPond Code**: Use Abjad or python-ly to create .ly files from user input (e.g., notes, rhythms).
2. **Compile to MIDI**: Run the bundled LilyPond executable via subprocess to output MIDI.
3. **Render to Audio**: Convert MIDI to WAV using libraries like mido (for MIDI handling) and an external synthesizer (e.g., FluidSynth via subprocess). Mix with pydub or similar.
4. **App Packaging**: Use tools like PyInstaller to bundle the app, including dependencies.

Example with python-ly for manipulation:
```python
from ly.document import Document
from ly.pitch.transpose import Transposer

doc = Document.load("input.ly")
transposer = Transposer(0, 2)  # Transpose up a major second
# Apply to music nodes...
doc.save("output.ly")
```
Then compile as above.
